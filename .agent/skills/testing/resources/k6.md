# 📉 Load Testing (k6)

**Context:** Use this to verify scalability before big releases.

## 1. The Script Structure
Tests reside in `tests/performance/`.
```javascript
import http from 'k6/http';
import { check, sleep } from 'k6';

export const options = {
  vus: 50, // Virtual Users
  duration: '30s',
  thresholds: {
    http_req_duration: ['p(95)<500'], // 95% of requests must be faster than 500ms
  },
};

export default function () {
  const res = http.get('https://my-app.cfapps.eu10.hana.ondemand.com/odata/v4/Invoices');
  check(res, { 'status is 200': (r) => r.status === 200 });
  sleep(1);
}
```

## 2. Scenarios
- **Smoke Test:** Minimal load (1 VU) to ensure the script works.
- **Load Test:** Expected normal load (e.g., 50 VUs).
- **Stress Test:** Breaking point (e.g., 500 VUs) to see how it fails.

## 3. Best Practices
- **Test Data:** Do not use production data. Use a script to pre-seed the DB if necessary.
- **Headers:** Remember to pass Authorization headers (JWT) if testing protected endpoints.

## 4. Analysis
- Focus on `http_req_duration` (Latency) and `http_req_failed` (Error Rate).
- Any error rate > 1% is usually a failure.
