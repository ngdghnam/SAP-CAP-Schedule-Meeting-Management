# 🚀 SAP CAP Agent Team: The Ultimate Prompt Guide

This guide provides **50 Best Practice Use Cases** (10 per Agent) to help you maximize the potential of your AI Team.

---

## 🏛️ 1. Solution Architect (The City Planner)
*Best for: High-Level Design, Integration Strategy, Business Alignment*

1.  **Project Initiation:** "Act as the Solution Architect. Interview me to define the high-level Business Process for a 'Vendor Portal' on BTP. Then produce a Business Requirement Specification (BRS)."
2.  **Clean Core Strategy:** "We need to extend the S/4HANA 'Purchase Order' process. Design a Side-by-Side extensibility pattern that keeps the core clean. Should we use SAP Event Mesh?"
3.  **Legacy Migration:** "Plan the migration of our neo-based Java app to Cloud Foundry CAP. Outline the database migration steps for moving from HANA 1.0 to HANA Cloud."
4.  **Integration Patterns:** "Compare 'API-Led' vs 'Event-Driven' integration for syncing Customer Data from Salesforce to SAP S/4HANA. List the pros/cons for our specific latency requirements."
5.  **Data Privacy (GDPR):** "Review our High-Level Design. Identify where PII data is stored and define a data retention and purging strategy using SAP ILM."
6.  **Technology Selection:** "Research the trade-offs between using SAP Fiori Elements vs. a Custom React App for our 'Warehouse Scanner' use case. Consider offline capabilities."
7.  **Security Architecture:** "Design the Principal Propagation flow from our Standalone App Router -> CAP Service -> On-Premise S/4HANA. Which Cloud Connector configurations are needed?"
8.  **Cost Estimation:** "Estimate the TCO (Total Cost of Ownership) for running this microservice on BTP. Include costs for HANA Cloud, Application Runtime, and Bandwidth."
9.  **Feasibility Study:** "Can we use the 'SAP Document Management Service' to store 5TB of video files? Check the technical limitations and pricing tier impact."
10. **Refactoring Roadmap:** "Our monolith CAP service has grown too large. Propose a plan to break it down into 3 microservices (Orders, Billing, inventory) using Domain-Driven Design."

## ⚙️ 2. Lead CAP Architect (The Engine Builder)
*Best for: Backend Logic, Schema Design, API Standards*

1.  **Schema Design:** "Draft the `db/schema.cds` for a 'Ride Sharing' module. Include Entities for Drivers, Passengers, and Trips. Use standard Aspects (`managed`, `cuid`)."
2.  **Service Definition:** "Create the `srv/service.cds`. Expose the 'Trips' entity but restrict sensitive fields. Add a custom Action `calculateFare`."
3.  **Complex Business Logic:** "Implement the `on('calculateFare')` handler in TypeScript. It needs to calculate distance between coords and apply surge pricing logic."
4.  **Performance Tuning:** "Analyze this CQL query: `SELECT * FROM Orders`. It's slow. Refactor it to use a HANA View or explicit Column selection to reduce memory usage."
5.  **Type Safety Refactor:** "Scan `srv/handlers/OrderHandler.ts`. Replace all instances of `any` with strict types generated by `cds-typer`."
6.  **External API Consumption:** "Write a wrapper function to call the S/4HANA `API_BUSINESS_PARTNER` using `sap-cloud-sdk`. Handle connectivity and destination retrieval."
7.  **Async Processing:** "The 'End of Month' report takes 30 seconds. Move this logic to a background job using CAP's persistent queue (`cds.queued()`). Update the UI via WebSocket when done."
8.  **Validation Logic:** "Implement a `@before` handler for 'Create Invoice'. Ensure the 'DueDate' is at least 30 days after 'IssueDate'. Throw a user-friendly error if not."
9.  **Unit Testing:** "Write a Jest unit test for the `DiscountCalculator` class. Cover edge cases like 'Negative Price' and 'Zero Quantity'."
10. **Local Data Seeding:** "Generate 50 realistic 'Orders' in `db/data/my.app-Orders.csv` for local testing. Include edge cases like 'Status=Cancelled'."

## 🎨 3. Lead Frontend Engineer (The Pixel Perfect Architect)
*Best for: React/Fiori UI, State Management, Accessibility*

1.  **UI Component Scaffolding:** "Create a `CustomerCard` component using React and Tailwind. It must look exactly like a Fiori Object Card (Header, Avatar, Status Badge)."
2.  **State Management:** "Implement a 'Shopping Cart' store using Zustand. It needs to persist to `localStorage` and sync across tabs."
3.  **Complex Form Handling:** "Build a multi-step 'Wizard' form for Employee Onboarding. Use `react-hook-form` and validation schemas (Zod). Step 2 depends on Step 1 input."
4.  **OData Integration:** "Write a Custom Hook `useOrders` that fetches data from our CAP Service. Implement Server-Side Pagination, Filtering, and Sorting."
5.  **Mockup Verification:** "Compare this Screenshot [upload image] with the SAP Fiori Design Guidelines. List 3 violations regarding spacing and typography."
6.  **Performance Optimization:** "The 'Dashboard' page is laggy. Analyze the re-render cycles. Memoize the expensive Chart components using `useMemo` and `React.memo`."
7.  **Accessibility (A11y) Audit:** "Audit our `NavBar` component. Ensure it supports Keyboard Navigation (Tab, Enter) and has correct ARIA labels for screen readers."
8.  **Responsive Design:** "Refactor the 'DataGrid' component. On Mobile (<768px), it should transform from a Table into a Stacked Card List."
9.  **Theming:** "Implement a 'Dark Mode' toggle. Use CSS Variables / Tailwind `dark:` classes to switch standard Fiori colors to their dark equivalents."
10. **Error Boundaries:** "Wrap the main content area in an Error Boundary. If a component crashes, show a friendly 'Something went wrong' graphic instead of a white screen."

## 🧪 4. Lead QA Automation Engineer (The Safety Architect)
*Best for: E2E Testing, Load Testing, Debugging*

1.  **Test Strategy:** "Designing the QA Strategy for the 'Payment' feature. Define the mix of Unit, Integration, and E2E tests required for 100% confidence."
2.  **Playwright E2E Test:** "Write a Playwright script that flows: Login -> Search Product -> Add to Cart -> Checkout. Assert success message appears."
3.  **Negative Testing:** "Write a test case that tries to upload a 'Virus.exe' file to the Document Service. Ensure the backend rejects it with 400 Bad Request."
4.  **Load Testing (k6):** "Create a k6 script to hammer the 'Search API' with 1000 VUs (Virtual Users). We need to verify auto-scaling works."
5.  **Flaky Test Debugging:** "The `profile.spec.ts` fails 10% of the time. Analyze the logs. Is it a race condition waiting for the API? Add a smart waiter."
6.  **API Testing:** "Write a script to fuzz-test our OData API. Send malformed JSON and huge payloads to ensure the server handles them gracefully."
7.  **Data Setup:** "Write a `breeder` script that generates 10,000 records in the test database before the Load Test starts, so we test against realistic volume."
8.  **CI/CD Gatekeeping:** "Review the failed pipeline logs. Why did the 'Lint Check' pass but 'Build' fail? Pinpoint the exact line number."
9.  **Visual Regression:** "Set up a Visual Test for the 'Landing Page'. It should fail if the pixel difference is > 1% (e.g., CSS broke the layout)."
10. **Scenario Simulation:** "Simulate a 'Network Timeout' in the middle of a transaction. Verify that the Frontend shows a 'Retry' button and doesn't duplicate the order."

## 🛣️ 5. DevOps Platform Engineer (The Mechanic)
*Best for: CI/CD, Infrastructure, Deployment*

1.  **MTA Configuration:** "Add a new 'Redis' service to our `mta.yaml`. Bind it to the `srv` module and ensure the resource name follows convention."
2.  **Pipeline Construction:** "Write a GitHub Actions pipeline that triggers on Pull Requests. It must run `npm test`, `npm run lint`, and build the MTAR."
3.  **Deployment Scripts:** "Write a script to automate the Blue/Green deployment to Cloud Foundry. It should verify the health of the Green instance before switching routes."
4.  **Infrastructure as Code:** "Terraform our BTP Subaccount. Define the entitlements for HANA Cloud, Event Mesh, and XSUAA in a `.tf` file."
5.  **Log Analysis:** "Analyze these 'Out of Memory' crash logs. Is it the Node.js heap limit? Recommend a new `memory-quota` setting for `manifest.yml`."
6.  **Security Scanning:** "Integrate a Container Scan (trivy) into our pipeline. It should block the build if High Severity CVEs are found in our base image."
7.  **Secret Management:** "We are currently hardcoding the API Key. Refactor this to use the `CredStore` service or GitHub Secrets injected at build time."
8.  **HANA Optimization:** "The `npm install` for HANA `db-deployer` is too slow in CI. Configure caching for the `node_modules` to speed up the build."
9.  **Custom Domain Setup:** "Explain the steps to map `app.mycompany.com` to our BTP application. Which `routes` need to be changed in `mta.yaml`?"
10. **Availability Monitoring:** "Set up an Alert Notification. If the application crashes or CPU > 90% for 5 minutes, send a message to our Slack channel."

## 🤝 6. Team-Level Workflows (The Swarm)
*Best for: Complex features requiring Backend + Frontend + QA.*
**How to use:** Address the "Principal Architect" or the "Team" generically.

1.  **The "Flexible Request System" (Full Stack Feature):**
    > "Team, I need a metadata-driven request system. It should allow admins to define new request types (e.g., User, Asset, Travel) dynamically, and users to fill them out. We need a 'Form Builder' UI and a generic Backend."
    *   *Result:* Principal Plans -> Solution Designs Schema -> CAP Builds Generic Service -> Frontend Builds Form Engine.

2.  **The "Legacy Migration" (Strategy + Execution):**
    > "We are migrating from Neo (Java) to BTP (CAP). Analyze our existing `web.xml` and `persistence.xml`, then scaffold a new CAP project that replicates this logic."
    *   *Result:* Solution analyzes Legacy -> CAP scaffolds New Project -> DevOps adds pipelines.

3.  **The "Performance Overhaul" (Review + Fix):**
    > "The 'Order History' page takes 8 seconds to load. Conduct a full-stack audit and fix the bottlenecks."
    *   *Result:* QA measures Load -> CAP optimizes Query -> Frontend adds Caching.

4.  **The "New Module" (End-to-End):**
    > "Build a 'Supplier Onboarding' module. It needs a Fiori UI for registration, a CAP service for validation, and an Event Mesh integration to sync with S/4HANA."
    *   *Result:* Sequential execution across all roles.

5.  **The "Security Hardening" (Audit):**
    > "Team, preparing for Go-Live. harden the application. ensure zero hardcoded secrets, full principal propagation, and correct role assignments."
    *   *Result:* Solution Designs Security -> CAP implements XSUAA -> DevOps configures BTPDestination.

## 🏭 7. Supply Chain & Procurement Special (The SCM Suite)
*Best for: Deep-dive scenarios in Supplier Management, Sourcing, and Logistics.*
**Context:** Use these prompts to build your "Supplier 360" Platform.

### 📦 Supplier Management & Portal
6.  **"The Supplier Registration Flow"**
    > "Team, build a public-facing Registration Form for new suppliers. It needs Captcha protection. Upon submission, trigger a workflow for 'Category Manager' approval. If approved, create a Business Partner in S/4HANA via API."
7.  **"The Certificate Expiry Tracker"**
    > "We need a background job that checks Supplier Certificates (ISO 9001). If expiring in 30 days, send an email to the Supplier and show a 'Warning' badge on their Portal Dashboard."
8.  **"The Supplier Profile Sync"**
    > "Design an Event-Driven pattern. When a Supplier updates their Address in S/4HANA, our BTP app must update its local cache immediately. Configure the SAP Event Mesh subscription."
9.  **"The Document Center"**
    > "Build a 'Legal Documents' repository in the Supplier Portal. Suppliers must upload NDAs (PDF only). Store files in SAP Document Management Service (DMS) and link IDs to the CAP Supplier entity."

### 📊 Qualification & Evaluation
10. **"The Dynamic Qualification Questionnaire"**
    > "Team, similar to the 'Flexible Request System', we need a 'Qualification Survey' engine. Category Managers should define questions (Text, Yes/No, File) for specific commodities (IT vs. Chemicals). Suppliers answer on the portal."
11. **"The Supplier Scorecard"**
    > "Build a 360-degree View. Fetch 'Hard Facts' (On-Time Delivery %) from S/4HANA and merge them with 'Soft Facts' (Auditor Review) from our BTP DB. Display as a Radar Chart."
12. **"The Audit Mobile App"**
    > "We need a mobile-first view for Site Auditors. They walk around the factory and tick checkboxes. It must work Offline and sync when back online."
13. **"The Risk Alert System"**
    > "Integrate an external API (Google News / Refinitiv) to monitor our Top 50 Suppliers. If 'Bankruptcy' or 'Strike' is mentioned, trigger a 'Critical Alert' to the Supply Chain Director."

### 💰 Sourcing & RFQ
14. **"The RFQ Wizard"**
    > "Create a 'Request for Quotation' wizard. Internal users select items from the Material Master (S/4 import). The system then generates a unique link to send to 5 invited suppliers."
15. **"The Sealed Bid System"**
    > "Implement a 'Sealed Bid' logic. Suppliers submit prices, but the Buyer cannot see them until the 'Deadline' expires. Ensure crypto/hashing is used for the bid amounts in the DB."
16. **"The Bid Comparison Tool"**
    > "Build a complex Analytical Page. Compare 5 Supplier Bids side-by-side. Highlight the 'Best Price' in Green and 'Longest Lead Time' in Red. Allow export to Excel."
17. **"The Interactive Negotiation Room"**
    > "Build a real-time Chat Interface between Buyer and Supplier contextually linked to an RFQ. Messages must be saved for audit trails."

### 🚚 Ordering & Logistics
18. **"The PO Confirmation Workflow"**
    > "Suppliers need to Confirm, Reject, or Propose Changes to a Purchase Order. If they change the 'Delivery Date', it must trigger a 'Variance Workflow' for the Buyer to accept."
19. **"The ASN Generator (Advance Shipping Notification)"**
    > "Allow Suppliers to flip a PO into an ASN. They select items + quantities. The system must validate total quantity <= PO quantity. Generate a PDF Packing Slip with a QR Code."
20. **"The Goods Receipt Variance Report"**
    > "Compare the ASN (Supplier said X) vs. the Goods Receipt (We counted Y). Show a discrepancy report and trigger a 'Debit Note' proposal if Y < X."
21. **"The JIT/JIS Dashboard"**
    > "For Just-In-Time delivery, we need a live WebSocket dashboard showing Trucks en route. Integrate with a mock TPS (Transport Provider System) API."

### 🧾 Invoicing & Finance
22. **"The Self-Billing Engine"**
    > "For trusted suppliers, we auto-generate the Invoice based on the Goods Receipt. Build a job that runs nightly, aggregates GRs, and creates a 'Self-Billing Invoice' PDF."
23. **"The Invoice OCR Bot"**
    > "Suppliers email PDF invoices. Use SAP Document Information Extraction (DOX) to extract Header/Line items. CAP Logic must try to '3-Way Match' (PO -> GR -> Invoice) automatically."
24. **"The Payment Status Tracker"**
    > "Suppliers keep asking 'When will I be paid?'. Expose a read-only view of the S/4HANA 'Payment Run' status (Clearing Document) on the Portal."
25. **"The Dispute Resolution Flow"**
    > "If an Invoice is blocked, allow the Supplier to open a 'Dispute Ticket'. Upload supporting evidence. Route to the AP (Accounts Payable) team."

### 🛡️ Quality Management (PPAP & APQP)
26. **"The PPAP Submission Wizard"**
    > "Team, build a Level 3 PPAP Workflow. The Supplier must upload 18 documents (Design Records, DFMEA, PFMEA, Control Plan, etc.). Validate that the 'Part Submission Warrant (PSW)' is signed before allowing final submit."
27. **"The Digital Control Plan"**
    > "Instead of uploading a PDF, build a 'Digital Control Plan' editor. Suppliers define Process Steps, Characteristics (Product/Process), and Evaluation Methods in a structured grid. Version control is critical."
28. **"The Deviation/Waiver Workflow"**
    > "Allow Suppliers to request a temporary 'Deviation' (e.g., use alternative material for Batch X). It must route to the Quality Engineer for approval and automatically expire after 3 months or 1000 units."
29. **"The 8D Problem Solving Report"**
    > "If a defect is found, trigger an '8D Report' task for the Supplier. They must fill out D1 (Team) to D8 (Congratulate Team). Enforce 'Root Cause Analysis' (5 Whys) in D4."
30. **"The Sampling Plan Calculator"**
    > "Build a utility that calculates the required sample size based on AQL (Acceptable Quality Limit) standards (ISO 2859-1). The Supplier enters Lot Size, and the system tells them how many parts to measure."
