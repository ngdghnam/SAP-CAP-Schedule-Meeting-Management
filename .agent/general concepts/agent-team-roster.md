### 🔴 Rules (The Law)
*Strict Checklists. Violating these causes immediate failure.*
*   **[Coding Standards](../rules/coding-standards.md)**: No `any`, No `console.log`, Semantic Versioning.
*   **[Git Workflow](../rules/git-workflow.md)**: Conventional Commits, Branch naming (`feature/*`).
*   **[Security Basics](../rules/security-basics.md)**: No hardcoded secrets, Input Validation required.

### 🔵 Guidelines (The Textbook)
*Detailed explanations and best practices.*
*   **[Backend Guidelines](../guidelines/backend-guidelines.md)**: How to layer your architecture (Service -> Handler -> Processor).
*   **[Design Guidelines](../guidelines/design-guidelines.md)**: How to make React look like Fiori (Tailwind Tokens).
*   **[Testing Guidelines](../guidelines/testing-guidelines.md)**: When to write Unit vs. Integration vs. E2E tests.
